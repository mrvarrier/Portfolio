UX Case Study - Zenodo Integration

TEAM AND ROLE

I led the UX design for this critical integration project at Correlate, working closely with UX researchers, developers, and product managers. As the lead designer, I conducted hands-on API research, designed the complete user experience, and created innovative solutions to overcome architectural constraints. My role expanded to include technical research when development resources were unavailable, making me the bridge between technical feasibility and user needs.

TIME FRAME

8 weeks (Critical sprint delivery)

Project Overview
Integration architecture showing Correlate's cloud-only platform connecting to Zenodo repository system through innovative server-mediated transfer

OVERVIEW

Seamless archival integration that transforms technical constraints into innovation opportunities.

Zenodo Integration enables European research organizations to archive their work directly from Correlate's cloud-only platform to the Zenodo repository system. The solution includes smart metadata pre-population, flexible archival workflows, and a server-mediated transfer system that maintains our architectural principles while delivering seamless functionality. Users can archive individual files or entire projects with minimal manual input, making compliance and long-term preservation effortless.

CONTEXT

Urgent integration requirement with seemingly impossible technical constraints.

Two European research organizations (FeMaLe and Cities2030) had committed to seamless archival capabilities as part of their funding proposals. With an 8-week deadline, we faced a fundamental challenge: Correlate's cloud-only architecture stores no files locally—only links to users' external storage like Google Drive. However, Zenodo's repository system requires direct file uploads. This created an architectural conflict that demanded innovative thinking to resolve without compromising our platform's core principles.

TECHNICAL RESEARCH & DISCOVERY

Hands-on API Investigation

With development resources unavailable, I personally tested Zenodo's REST API endpoints, mapped mandatory metadata fields, and documented authentication requirements. This technical research became the foundation for all design decisions and revealed critical constraints early in the process.

Architecture Constraint Analysis

I discovered the fundamental conflict between Correlate's cloud-only approach and Zenodo's file upload requirements. This analysis was crucial for understanding the technical boundaries within which the user experience had to be designed.

User Workflow Research

Collaborated with UX researchers to understand how each organization approaches archival workflows. We identified different compliance needs and process preferences that required flexible yet unified solutions.

KEY DISCOVERY #1

Technical constraints became innovation catalysts. The server-mediated transfer solution emerged from embracing our architectural limitations rather than working around them, resulting in a more elegant and secure approach.

KEY DISCOVERY #2

Users needed different archival pathways: quick individual file archival for routine work and comprehensive bulk project archival for milestone compliance. A one-size-fits-all approach wouldn't meet their diverse workflow needs.

KEY DISCOVERY #3

Metadata pre-population from existing Correlate data could significantly reduce user effort while maintaining accuracy. Document titles, creation dates, and author information were readily available and could eliminate redundant data entry.

PROBLEM STATEMENT

"How do we enable seamless archival from a platform that stores no files locally to a repository system that requires direct file uploads—all while maintaining our architectural principles and meeting urgent deadlines?"

SOLUTION DESIGN & INNOVATION

Integration Setup Interface
Clean, guided setup flow for connecting Zenodo accounts with clear authentication steps and permission explanations

Seamless Integration Setup

I designed a straightforward setup flow that guides users through Zenodo account connection without overwhelming them with technical details. The interface clearly explains what permissions are needed and why, building trust while maintaining security. Users can connect their accounts once and then archive seamlessly throughout their workflow.

Server-Mediated Transfer Innovation

The breakthrough solution maintains our cloud-only architecture while enabling file-based archival. When users initiate archival, our servers temporarily download files from their cloud storage, transfer them to Zenodo via API, then immediately delete local copies. This approach preserves our architectural principles while delivering the functionality users need, creating a secure and elegant solution to an impossible constraint.

Technical Architecture
Flow diagram: User's Cloud Storage → Correlate Server (temporary) → Zenodo Archive → Immediate Deletion
Archival Selection Interface
Intuitive file and project selection with bulk operations, smart filtering, and clear archival status indicators

Flexible Archival Workflows

Understanding that different organizations have different archival needs, I designed dual pathways: quick individual file archival for routine documentation and comprehensive bulk project archival for compliance milestones. Users can select multiple files, entire folders, or complete projects with a few clicks, making archival as simple as possible regardless of scope.

Smart Metadata Pre-population

Rather than forcing users to manually recreate information that already exists in Correlate, I designed the system to automatically extract document titles, creation dates, author information, and project associations. This significantly reduces form completion time while ensuring accuracy and consistency across archived materials.

Metadata Form Design
Pre-populated form with smart defaults, validation feedback, and draft/publish options for collaborative review

Draft & Publish Flexibility

Recognizing that archival often involves collaborative review, I included options for users to save drafts for team feedback or publish immediately when working independently. This flexibility accommodates different organizational approval processes while ensuring compliance requirements are met consistently.

Status Tracking & Confirmation

The final piece of the user experience provides clear feedback throughout the archival process. Users can see upload progress, receive confirmation when archival is complete, and access direct links to their archived materials in Zenodo. Error handling is graceful, with clear explanations and suggested actions if any step encounters issues.

Progress & Confirmation UI
Status indicators, progress tracking, success confirmations, and error handling with recovery options

RESULTS AND IMPACT

Successfully delivered a working MVP within the critical 8-week deadline, enabling both European research organizations to meet their proposal commitments. The integration seamlessly bridges Correlate's cloud-only architecture with Zenodo's file-based requirements without compromising either system's principles.

The server-mediated transfer approach became a reusable architectural pattern for future integrations, demonstrating how technical constraints can drive innovation. Users can now archive their research materials with minimal effort while maintaining compliance with European research data requirements.

This project significantly enhanced my skills in technical UX design, teaching me the value of hands-on technical research as a foundation for design decisions. Working within extreme constraints while maintaining user experience quality proved that limitations often produce more creative and elegant solutions than unlimited freedom.

KEY LEARNINGS

TECHNICAL RESEARCH DRIVES DESIGN

Hands-on API investigation prevented impossible design decisions and revealed creative solution opportunities. Understanding technical constraints early allowed me to design within realistic boundaries while pushing for innovation.

CONSTRAINTS SPARK CREATIVITY

The server-mediated transfer solution emerged from embracing our architectural limitations. Working within constraints often produces more elegant solutions than unlimited freedom, forcing creative problem-solving that benefits both users and systems.

STRATEGIC TEAM COLLABORATION

Clear role division with design leadership accelerated delivery without compromising quality. When technical resources are limited, UX designers can step into technical research roles to maintain project momentum.
