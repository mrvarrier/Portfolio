01. OVERVIEW

Archival Integration for Research Compliance

European consortium partners (FeMaLe and Cities2030) faced strict EU Horizon 2020 compliance requirements to archive research data in approved repositories like Zenodo—but lacked seamless integration with their existing Correlate workflows. Our solution was a comprehensive archival integration that allows users to seamlessly transfer collaborative work from Correlate to Zenodo while meeting regulatory requirements and minimizing workflow disruption.

03. RESEARCH & DISCOVERY

Technical Constraints Meet User Needs

API Technical Investigation
With our development team fully occupied, I leveraged my computer science background to personally test Zenodo's API capabilities. I set up integration endpoints, documented mandatory fields, and identified technical limitations—creating comprehensive documentation that informed our design constraints.
User Research with Consortium Members
Our UX researcher conducted interviews with FeMaLe and Cities2030 consortium members to understand archival workflows, compliance requirements, and pain points. We identified opportunities to reduce administrative overhead while ensuring regulatory compliance.
Competitive Analysis & Systems Research
I researched existing archival integrations and Zenodo's interface patterns to understand user expectations. This revealed that most solutions required users to completely reconstruct their work in new platforms—highlighting our opportunity for seamless data transfer.
INSIGHT 01
Zenodo's API supports both draft and direct publication modes with different metadata requirements. This technical constraint needed careful balance with user workflow preferences to avoid overwhelming researchers.
INSIGHT 02
Consortium members preferred bulk archival for project deliverables rather than individual files, needing to archive entire project phases while maintaining organizational structure.
INSIGHT 03
The biggest pain point was data recreation—researchers spent hours manually uploading files and re-entering metadata that already existed in Correlate, creating resistance to compliance requirements.
⚠️
Critical Technical Discovery
During my API research, I uncovered a fundamental architecture challenge: Correlate doesn't store files locally—all documents exist as links to users' external integrations (Google Drive, etc.). This meant we couldn't directly transfer files to Zenodo using standard methods.
SOLUTION APPROACH
I proposed leveraging our existing server infrastructure to temporarily download files from users' Google Drive, transfer them to Zenodo via API, then immediately delete them from our servers. This solution maintained our architecture principles while enabling seamless archival functionality.

Problem Statement

"How might we enable European research consortium members to seamlessly archive their collaborative work from Correlate to Zenodo while meeting strict compliance requirements and minimizing workflow disruption?"

04. DESIGN PROCESS

Systematic Problem Solving

1

Technical Documentation & Team Alignment

After completing API research, I created comprehensive technical documentation outlining integration possibilities, constraints, and recommended approaches. This documentation served as the foundation for design decisions and development planning.

2

User Flow Mapping

I collaborated with the UX researcher to map detailed user journeys, identifying two primary scenarios: individual file archival and bulk project archival. These flows incorporated both technical constraints and user preferences discovered during research.

3

Low-Fidelity Prototyping

I created initial wireframes focusing on the archival workflow, metadata input forms, and integration authentication. These prototypes prioritized ease of use while ensuring all mandatory Zenodo fields were captured efficiently.

4

Stakeholder Review & Iteration

I presented initial designs to the full Correlate team, incorporating feedback on technical feasibility, user experience, and business requirements. Multiple review cycles ensured alignment between design vision and implementation reality.

5

Design Handoff & Collaboration

Our Chief Design Officer took over visual design refinement while I continued collaborating on interaction design and user flow optimization. This collaborative approach ensured design consistency while meeting our tight timeline.

05. KEY DESIGN DECISIONS

Solutions That Bridge Constraints

Simplified Integration Authentication

Designed a streamlined OAuth setup flow that allows users to connect their Zenodo accounts without leaving Correlate. The interface clearly communicates permissions and provides helpful guidance for first-time Zenodo users.

Smart Metadata Pre-population

Leveraged existing Correlate data to automatically populate Zenodo's mandatory fields where possible. Document titles, creation dates, and author information are extracted from Correlate, reducing form completion time by an estimated 60%.

Flexible Archival Options

Created two distinct archival pathways: individual file archival for quick uploads and bulk project archival for comprehensive deliverable packages.

Draft vs. Publish Workflow

Users can save drafts for later completion or publish immediately, accommodating different workflow preferences and compliance timelines while clearly communicating the permanence of published archives.
